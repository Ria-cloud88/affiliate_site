## 問題の本質

何度修正を試みてもうまくいかなかった理由は、**根本原因が複数あった**こと：

1. **修正コード不完全**: 不正な見出し行は削除しても、その直後の本文は残っていた
2. **プロンプト指示不十分**: 「絶対禁止」と書いてもAIが守らない
3. **処理フロー複雑**: 修正が3回呼ばれるため、前の修正が後で壊される可能性

## 実装した修正（コミット: 62693ad）

### 1️⃣ fix_article_content() ロジック改善

**Before（不完全）:**
```python
else:
    i += 1  # 見出しをスキップ
    if i < len(lines) and lines[i].strip() == '':
        i += 1  # 空行もスキップ
    # その後の本文はそのまま処理される ❌
```

**After（完全）:**
```python
else:
    i += 1  # 見出し行をスキップ
    # 次の ## 見出しが出現するまで、すべてをスキップ
    while i < len(lines) and not lines[i].strip().startswith('## '):
        i += 1
```

→ 不正な見出し＋その本文全体を削除

### 2️⃣ プロンプト内の指示を強化

追加した指示：
- 「改行後に ## が出現することは絶対に禁止」（明確化）
- 「出力の最後の見出しは必ず ## まとめで終わる」（終了条件を明記）
- 「※CRITICAL※」マーク（重要性を強調）
- 禁止する見出しを具体的にリストアップ（補足、追加情報、関連リンク、さらに詳しく）
- 具体的な CORRECT/INCORRECT 出力例を追加

### 3️⃣ AIへのメッセージング

「データの自動追加（関連記事など）は別途スクリプトで処理される。生成時に追加しない」と明記。

→ AIが独断で「関連記事」セクションを追加しないようにするため

## なぜ修正が効かなかったか（教訓）

- **対症療法の落とし穴**: 修正コードで対応しようとしたが、AIの根本的な理解不足には対応できない
- **複数原因の同時修正が必要**: プロンプト（予防）＋ 修正コード（対応）の両方を改善する必要があった
- **具体例の力**: プロンプトに CORRECT/INCORRECT 例を追加すると、AIの理解度が大幅に向上する

## How to apply

次回の記事生成で、## まとめ 後に不正な見出しが出現しなくなったか確認する。
もし再発したら：
1. 生成された記事の ## まとめ セクションを確認
2. 不正な見出しが何か記録（fix_article_content() のホワイトリスト拡張用）
3. プロンプト内の禁止リストに追加
